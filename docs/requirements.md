# 易拍（easy-photo）需求文档

## 1. 产品概述

易拍是一款为用户提供**拍照教程**的应用。用户可以按风格、姿势、色调等维度浏览他人分享的拍照教程，学习滤镜、调色、修图技巧；也可以上传自己的教程、收藏与点赞感兴趣的内容。系统分为**管理端**和**用户端**两部分。

- 英文名：easy-photo
- 核心价值：让用户快速找到并学习到合适的拍照与修图方法。

## 2. 角色与权限

| 角色 | 说明 | 主要权限 |
| --- | --- | --- |
| 游客 | 未登录访问者 | 浏览 public 教程、按分类筛选、查看教程详情 |
| 普通用户 | 自注册登录 | 游客全部权限 + 上传/编辑/删除自己的教程、设置可见范围、收藏、点赞、查看收藏页 |
| 管理员 | 独立后台账号 | 分类管理、用户管理、查看/下架/删除所有教程（含 private） |

说明：
- 用户可自行注册、登录（账号 + 密码），管理员另有独立后台入口。
- 教程遵循「作者自管 + 管理员全权」：作者可管理自己的教程，管理员可管理全部教程。

## 3. 分类模型（分类 + 维度，两张表）

分类采用两级结构，用两张表存储，分类可由管理员**自定义**，数量不固定：

- **分类表（category）**：一级，管理员可新增 / 删除。初始可预置风格、姿势、色调，但不限于这三个。
- **维度表（dimension）**：二级，每条维度归属一个分类（`category_id` 外键关联分类表）。

各分类下的维度示例：

- 风格：生活风、甜美风、忧郁风、自然风……
- 姿势：站立、坐姿、半身、蹲姿……
- 色调：冷暖色调、绿色、蓝色……

规则：
- 管理员可**新增 / 删除一级分类**，并可在各分类下**新增 / 删除维度**。
- **分类名称不可重复**（全局唯一）。
- **同一分类下的维度名称不可重复**（同分类内唯一，不同分类下可同名）。
- 一个教程可关联**多个维度**（通过教程-维度关系表维护），维度可跨越不同分类；不强制每个分类必选。
- 用户端可按任一分类下的维度独立筛选。
- 分类、维度均采用**逻辑删除**（软删除）：删除时标记为已删除而非物理移除，已引用的历史教程数据保持完整；唯一性约束仅在未删除记录范围内校验。

## 4. 教程结构

一个教程包含元数据和富文本正文两部分。

### 4.1 元数据
- 标题（必填）
- 封面图（必填，单张）
- 分类标签：可关联多个维度（每个维度归属某一分类），至少选择一个
- 可见范围：public / private
- 作者、创建时间、更新时间
- 点赞数、收藏数（统计字段）

### 4.2 正文（富文本）
- 正文为**富文本内容**，可混合包含文字和图片：
  - 文字支持基本排版（分点、加粗等）。
  - 可在正文中插入若干图片（可多张）。
- 富文本内容中的图片同样存储于服务器本地磁盘，正文保存图片的访问路径 / URL。

## 5. 用户端功能

### 5.1 分类浏览
- 分类以 **Tab** 形式展示：
  1. 选中一级分类（风格 / 姿势 / 色调）
  2. 继续选择该分类下的二级维度
  3. 展示该维度下的教程**卡片列表**（卡片含封面 + 标题）
- 可按任一分类维度独立筛选。

### 5.2 教程详情
- 展示元数据、富文本正文、作者信息。
- 提供点赞、收藏按钮。

### 5.3 可见性规则
- public：所有用户（含游客）可见。
- private：仅作者本人与管理员可见。
- 管理员可查看所有教程（含 private）。

### 5.4 上传与管理
- 登录用户可上传教程：填写标题、封面、分类维度（可多选）、富文本正文、可见范围。
- 作者可编辑、删除自己的教程。

### 5.5 收藏与点赞
- 用户可对某一教程进行**收藏**、**点赞**（可取消）。
- **收藏页**：展示当前用户收藏的全部教程。

## 6. 管理端功能

### 6.1 分类管理
- 管理员可自定义一级分类：新增、删除分类。
- 管理员可在各分类下新增、删除二级维度。

### 6.2 用户管理
- 管理员可新增、删除用户。

### 6.3 教程管理
- 管理员可查看所有教程（含 private）。
- 管理员可下架、删除任意教程。

## 7. 技术栈

### 7.1 前端
- Vue3 + Element Plus。
- 页面主色调为淡黄色，搭配莫兰迪色系、淡黄色或米白色，营造柔和清新的视觉风格。

### 7.2 后端
- Java 8 + Spring Boot + MySQL。
- 认证：使用 **JWT**。用户登录后签发 Token，后续请求通过请求头携带 Token 进行鉴权；服务端校验 Token 的有效性与角色权限。
- 密码加密存储（如 BCrypt）。

### 7.3 图片存储
- 图片（封面、步骤示例图）存储于**服务器本地磁盘**。
- 数据库仅保存图片的相对路径 / 访问 URL。

## 8. 数据模型概览

| 表 | 说明 | 关键字段 |
| --- | --- | --- |
| user | 用户 | id、username、password、role（user/admin）、deleted、created_at |
| category | 一级分类 | id、name（管理员自定义，唯一）、deleted、created_at |
| dimension | 二级维度 | id、category_id、name（同分类内唯一）、deleted、created_at |
| tutorial | 教程 | id、title、cover_url、content（富文本）、visibility、author_id、like_count、favorite_count、deleted、created_at、updated_at |
| tutorial_dimension | 教程-维度关系 | id、tutorial_id、dimension_id、deleted |
| tutorial_favorite | 收藏 | id、user_id、tutorial_id、deleted、created_at |
| tutorial_like | 点赞 | id、user_id、tutorial_id、deleted、created_at |

> 说明：教程与维度为**多对多关系**，通过 `tutorial_dimension` 关系表维护，教程本身不再持有固定的分类字段。`content` 存储富文本（HTML 或结构化 JSON，实现时确定），其中内嵌图片保存为指向本地磁盘的访问路径 / URL。
>
> 所有表均采用逻辑删除，统一通过 `deleted` 标记位区分；查询时默认过滤 `deleted = true` 的记录。名称唯一性约束（`category`、`dimension`）仅针对未删除的记录生效，避免与历史软删除记录冲突。
